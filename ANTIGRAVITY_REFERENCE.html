🎯 El Problema

❌ Token Expira

Sin mecanismo de refresh, el usuario queda atrapado:

  • 401 Unauthorized
  • localStorage.clear() borra TODO
  • window.location.reload() pierde estado
  • Usuario desincronizado

❌ Seguridad

Validación débil:

  • seller_id viene del frontend (no validado)
  • Sin límite de intentos de login
  • No hay separación auth/preferences

❌ Producción

No ready para deployment:

  • Assets servidos con rutas relativas frágiles
  • CORS sin restricciones
  • Sin retry logic para conexión inestable

✅ La Solución

🔐 Tokens Cortos + Refresh

Access token: 15 min | Refresh token: 7 días

  • Interceptor inteligente auto-refresh
  • Si refresh falla → logout seguro
  • Sin page reload innecesario

📦 Zustand Store

Single source of truth para la app:

  • useAuthStore: tokens + user
  • useUIStore: tema (independiente)
  • Persist automático, solo lo necesario

🚀 Producción Ready

Deploy seguro y escalable:

  • Railway backend + Vercel frontend
  • CORS restringido por dominio
  • Retry automático 3 intentos

🎓 Conceptos Clave a Entender

Zustand vs localStorage

❌ Antes:

localStorage.setItem('user', JSON.stringify(user))

✅ Después:

const { setUser } = useAuthStore() // Zustand persist automáticamente

Tokens Cortos + Refresh

❌ Antes: 1 token 30 días = hacked por 30 días

✅ Después:

  • Access: 15 min (bajo riesgo)
  • Refresh: 7 días (guardado seguro)
  • Si expira → auto-refresh

Interceptor Inteligente

❌ Antes: Request 401 → logout

✅ Después:

  • Token expirado → auto-refresh
  • Reintenta request
  • Realmente inválido → logout
